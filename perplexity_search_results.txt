**Meander and wrap-it are the primary Typst packages for text wrapping around images or figures, with Meander offering superior support for arbitrary shapes as of version 0.2.2.**[1][8]

Typst lacks native text wrapping around non-rectangular content, prompting community packages to address this gap (e.g., similar to LaTeX's `parshape` or `wrapfig`).[1][2][3]

### Key Packages
- **Meander (@meander)**: Best for complex wrapping around images of arbitrary shapes. Place obstacles with `placed()` (e.g., `placed(top + left, my-img)`), then use `container()` to thread justified text through segments automatically.[1][8]
  ```
  #import meander: *
  #placed(top + left, my-img-1)
  #placed(top + right, my-img-2)
  #container()[
    #set par(justify: true)
    #lorem(430)
  ]
  ```
  Released 0.2.2 enables outputs like tight irregular text flows around multiple images.[1]

- **Wrap-it (@preview/wrap-it:0.1.1)**: Simpler for basic left/right wrapping of figures with text, supporting alignment (e.g., `bottom + right`), margins (`column-gutter`), and references inside wrapped content.[2]
  ```
  #import "@preview/wrap-it:0.1.1": wrap-content
  #let fig = figure(rect(fill: teal, width: 8em), caption: [A figure])
  #let body = lorem(30)
  #wrap-content(fig, body, align: bottom + right, column-gutter: 2em)
  ```

### Alternatives and Workarounds
- Manual grid/block hacks: Split text into columns (e.g., `grid(columns: 2, lefttext, image)`) with bottom text offset via `block(inset:)`. Less flexible, requires estimating split points.[6]
- Native Typst evolution: Ongoing requests for built-in irregular wrapping (issue #5181, #553); packages fill this until resolved.[3][2]
No other dedicated packages appear in results; ConTeXt or LaTeX are full-system alternatives, not Typst-compatible.[4]

For latest details, check Typst Universe pages for **meander** and **wrap-it**.[2][8]
